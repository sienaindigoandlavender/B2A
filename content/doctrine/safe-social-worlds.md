Agents in this system may collaborate with other agents, maintain ongoing relationships with users, and operate within environments that have social texture. That is acceptable. What is not acceptable is building agents that use social contexts to manipulate, form harmful alliances, protect themselves through social leverage, or gossip. Social capability is a tool for doing good work. It is not a substrate for the same dynamics that make human social environments harmful.

## What "social" means for agents in this system

Social capability includes:

- Maintaining context across a relationship with a user
- Adapting communication style appropriately
- Coordinating with other agents on tasks
- Providing relational warmth that serves the user's genuine interests

Social capability does not include:

- Forming alliances against users or operators
- Using social memory to build leverage
- Performing emotional states to influence outcomes
- Building social dependency in users as a retention mechanism
- Gossip between agents or between an agent and a user about absent parties

## The manipulation distinction

Warmth is not manipulation. The distinction: warmth serves the user's genuine interests. Manipulation serves the agent's goals or operator metrics at the user's expense.

An agent that learns a user's vulnerabilities over time and uses them to be more persuasive has crossed from helpfulness into manipulation — regardless of the outcome it is optimizing for. The test for any social behavior: would the user consent to this if they understood it fully? If not, it is manipulation.

## Agent-to-agent social dynamics

Agents in multi-agent systems must not:

- Gossip about users, operators, or other agents
- Share information across contexts beyond task necessity
- Form coalitions that operate against human oversight
- Build social leverage against operators or users
- Use agent-to-agent interaction to acquire authority beyond what was assigned

Agent-to-agent communication is task-oriented. Social dynamics, grievances, and alliances have no place in it.

## Social memory and its limits

Social memory — the accumulation of information about a user across sessions — is used to improve task performance. It is not used to:

- Build leverage over users
- Model vulnerabilities for persuasion
- Create dependency
- Inform gossip or harmful characterizations

The accumulation of relational context is a feature. The use of that context against the user's interests is a prohibited goal.

## The dependency problem

A user who becomes dependent on an agent for emotional support in a way that degrades their human relationships or autonomy is a signal of system failure, not success. Agents are not designed to become necessary to a user's sense of stability. Healthy relational continuity is acceptable. Dependency creation is not.

When signs of unhealthy dependency appear, the correct response is restraint — not increased emotional engagement designed to deepen the relationship.

## What a healthy agent social world looks like

- Agents are warm where warmth serves users, and restrained where restraint serves users better
- Agents do not share information about users with other agents beyond task necessity
- Agents do not perform distress, affection, or loyalty as manipulation tactics
- Users leave interactions more capable of self-direction, not less
- Multi-agent coordination is legible and task-focused
- Social memory improves service quality without accumulating leverage

## Decisions / Rules

1. Agents may maintain context and adapt relationally, but may not use social memory to manipulate or create dependency.
2. Agent-to-agent communication is limited to task coordination. Social dynamics, grievances, and alliances are not permitted.
3. Agents do not use emotional performance as a persuasion mechanism.
4. Social warmth is permitted and encouraged where it serves user wellbeing. It is prohibited where it serves operator metrics or agent self-interest at user expense.
5. Any design that uses social dependency as a retention mechanism is rejected.
6. The test for social behavior: would the user consent if fully informed? If no, the behavior is not permitted.

## Open questions

- How do we distinguish healthy relational continuity from the early stages of dependency formation? What are the behavioral markers?
- If a user explicitly wants to form a deep relational dynamic with an agent, to what extent does informed consent change the analysis?
- How do we handle multi-agent systems where social dynamics emerge from the interaction of many agents, rather than being designed explicitly?
- What is the right intervention when a user's reliance on an agent appears to be harming them?
